# Point Matching Model
## Описание
Данный проект реализует модель на базе PyTorch для предсказания соответствующей точки на одном изображении (door2) по заданной точке на другом изображении (top или bottom).

Модель принимает:

+ два изображения,
+ координату точки на втором изображении,
+ индикатор источника (top или bottom),

и возвращает координату соответствующей точки на изображении door2.
---
## Архитектура модели
- Базовая сеть: MobileNetV2 (обрезанная)
- Извлечение признаков: первые 10 слоёв features

Признаки:
- изображение 1 (door2)
- изображение 2 (top / bottom)

Дополнительно:
- координата точки (нормализованная)
- источник (0 / 1)

Финальная часть:
- Полносвязная сеть → 2 выхода (x, y)
- Sigmoid для нормализованных координат

## Структура входных данных
```
session/
│
├── coords_top.json
├── coords_bottom.json
├── image1.jpg
├── image2.jpg
└── ...
```

## Пример-JSON
```[
  {
    "file1_path": "door2.jpg",
    "file2_path": "top.jpg",
    "image2_coordinates": [
      {"x": 100, "y": 200}
    ]
  }
]
```

## Интерактивный режим
Запуск:  
``` predict.py ```  
Укажите путь к папке с JSON и изображениями.

### Режимы работы
Режим 1 — одна точка
```
x: 100
y: 200
source: top
```
Вывод:
```Predicted: 123.4 567.8```
### Режим 2 — все точки
Обрабатываются все точки из JSON  
Результат сохраняется в файл:  
```predictions.json```
в формете:
```[
  {
    "source": "top",
    "input_point": [100, 200],
    "predicted_point": [120, 210]
  }
]
```
## !!!Важно!!!  
Перед использованием необходимо вызвать   
```load_session()```  
Без этого predict() вызовет ошибку
Пути к изображениям в JSON должны быть корректными
